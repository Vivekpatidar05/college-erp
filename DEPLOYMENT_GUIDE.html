Deployment Guide

Step-by-step instructions to connect MongoDB Atlas and deploy on Railway — from zero to live URL.

Overview MongoDB Atlas GitHub Railway Deploy Environment Vars Verify Local Dev Troubleshoot

0 Architecture Overview

Your EduCore ERP runs as three components:

🚂
Railway
Hosts Flask app via Gunicorn. Auto-deploys from GitHub.
🍃
MongoDB Atlas
Cloud database. Free M0 tier — 512 MB storage.
🐙
GitHub
Source code. Railway watches this for auto-deploys.
💡
Railway free tier gives you $5/month credits — enough for a small Flask app. Atlas M0 is permanently free.

1 Set Up MongoDB Atlas

1
Create a free account
Go to cloud.mongodb.com → Sign Up (free). Verify your email.
2
Create a new Project
Click New Project → name it EduCore → Create.
3
Build a Free Cluster (M0)
Click Build a Database → choose M0 FREE tier → Select your nearest region (e.g. AWS Mumbai) → Name it Cluster0 → Create.
4
Create a Database User
In the Security Quickstart: choose Username and Password → set username e.g. educore_user → generate a strong password → Create User. Save the password — you'll need it.
5
Allow all IP addresses (for Railway)
In Network Access → Add IP Address → click "Allow Access from Anywhere" → enter 0.0.0.0/0 → Confirm. (Railway uses dynamic IPs, so this is required.)
6
Get your Connection String
Go to Clusters → Connect → Drivers → choose Python / 3.12 or later → copy the connection string. It looks like: 
# Your Atlas connection string (replace <password> and <dbname>)
mongodb+srv://educore_user:<password>@cluster0.abc12.mongodb.net/?retryWrites=true&w=majority
⚠️
Replace <password> with the password you set in Step 4. Do NOT include angle brackets. The database (college_erp) will be created automatically when the app first writes to it.

2 Push Code to GitHub

📦
Railway deploys directly from GitHub. You must push your code there first. Never commit your .env file — the .gitignore already excludes it.

Open terminal in your project folder

# Navigate to the project
cd college_erp

# Initialize Git repository
git init
git add .
git commit -m "Initial commit – EduCore ERP with auth"

Create GitHub repo and push

# Create a new repo at github.com first (no README), then:
git remote add origin https://github.com/YOUR_USERNAME/college-erp.git
git branch -M main
git push -u origin main
✅
Your code is now on GitHub. From here, every git push will trigger an automatic redeploy on Railway.

3 Deploy on Railway

1
Sign up / log in to Railway
Go to railway.app → Login with GitHub (recommended for auto-deploys).
2
Create a New Project
Click New Project → Deploy from GitHub repo → authorize Railway → select your college-erp repo.
3
Railway auto-detects Python
Railway reads your Procfile and requirements.txt automatically. It will install deps and start Gunicorn. Wait for the build to finish (1–2 min).
4
Add Environment Variables
Go to your service → Variables tab → add all variables from the table below. This is the most important step.
5
Generate a Public URL
Go to Settings → Networking → click Generate Domain. You'll get a URL like college-erp-production.up.railway.app.
6
Redeploy after adding variables
After adding env vars, Railway auto-redeploys. Watch the Deployments tab — it should show Success in green.

4 Environment Variables (Railway)

Go to Railway → your service → Variables tab → add each of these:

VariableValueRequired?
MONGO_URI Your full Atlas connection string
mongodb+srv://user:pass@cluster0.xxxx.mongodb.net/?retryWrites=true&w=majority		 Required
SECRET_KEY A long random string. Generate one:
python3 -c "import secrets; print(secrets.token_hex(32))"		 Required
DB_NAME college_erp Optional (default: college_erp)
PORT Auto-set by Railway Auto
🔐
Never put real secrets in code or push them to GitHub. Only set them in Railway's Variables panel. The .env file is for local dev only and is git-ignored.

5 Verify the Live App

  1. Open your Railway URL (e.g. https://college-erp-production.up.railway.app)
  2. You should see the EduCore login page
  3. Log in with admin / admin123 — the default admin is auto-created on first boot
  4. Go to User Management and change the admin password immediately
  5. Add a student and teacher, then create accounts for them to verify all three login roles work
🎉
If you see the dashboard — you're live! MongoDB Atlas is connected and all data will persist in the cloud.

6 Local Development with .env

For local testing (without Railway), use the .env.example file:

# 1. Copy the example file
cp .env.example .env

# 2. Edit .env and fill in your Atlas URI and secret key
nano .env

# 3. Install dependencies
pip install -r requirements.txt

# 4. Run locally
python app.py
💡
python-dotenv is already in requirements.txt. Flask will auto-load your .env file so all env vars are available locally.

Generate a strong SECRET_KEY

python3 -c "import secrets; print(secrets.token_hex(32))"
# Example output: a3f8c2e1d9b6f4a7c0e5b2d8f1a6c9e3b4d7f0a2c5e8b1d4f7a0c3e6b9d2f5

7 Troubleshooting

App crashes on Railway

Check Deploy Logs in Railway dashboard. Common causes:

  • Missing MONGO_URI — variable not set in Railway Variables tab
  • Wrong Atlas password — special chars in password must be URL-encoded (e.g. @ → %40)
  • IP not whitelisted — ensure 0.0.0.0/0 is in Atlas Network Access

Authentication error from MongoDB

# Bad: password contains special characters not URL-encoded
mongodb+srv://user:p@ss!word@cluster...

# Good: URL-encode special chars
mongodb+srv://user:p%40ss%21word@cluster...

Port binding error

The Procfile uses $PORT which Railway sets automatically. Never hardcode a port number.

Changes not deploying

git add .
git commit -m "your change description"
git push origin main
# Railway auto-deploys on every push to main

Reset admin password

If you're locked out, connect to Atlas via Compass or the web shell and run:

# In MongoDB Atlas Data Explorer → college_erp → users collection
# Find the admin document and update the password field with a new hash.
# Or delete the admin document — the app will recreate it on next boot.
