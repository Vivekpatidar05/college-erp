0 Architecture

🎓
DSVV ERP
Flask + Gunicorn — Python web app with 15+ modules
🍃
MongoDB Atlas
Cloud database — Free M0 tier. All ERP data stored here.
⚡
Render
Free web hosting. Auto-deploys from GitHub on every push.
ℹ️Render Free tier gives 750 hrs/month. MongoDB Atlas M0 is permanently free (512 MB). Both are sufficient for a college ERP demo.

1 MongoDB Atlas Setup

1
Create free account
Go to cloud.mongodb.com → Sign Up → verify email.
2
Create new project
Click New Project → name it DSVV-ERP → Create Project.
3
Build free cluster (M0)
Build a Database → M0 FREE → choose region nearest to you (e.g. AWS Mumbai ap-south-1) → Name: Cluster0 → Create.
4
Create database user
Security Quickstart → Username/Password → username: dsvv_admin → generate strong password → Create User. Save the password!
5
Allow all IPs (for Render)
Network Access → Add IP Address → Allow Access from Anywhere → 0.0.0.0/0 → Confirm. Render uses dynamic IPs, so this is required.
6
Get connection string
Clusters → Connect → Drivers → Python / 3.12+ → copy the string.
# Your Atlas URI (replace <password> with actual password — no angle brackets)
mongodb+srv://dsvv_admin:<password>@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority
⚠️Special characters in password (@ ! # $ etc.) must be URL-encoded. Use urlencoder.org to encode just the password part.

2 Push to GitHub

🔐Never commit .env to GitHub — your .gitignore already excludes it.

Initialize & push

# In your project folder (dsvv_erp/)
git init
git add .
git commit -m "Initial commit — DSVV ERP full system"

# Create new repo at github.com (no README), then:
git remote add origin https://github.com/YOUR_USERNAME/dsvv-erp.git
git branch -M main
git push -u origin main
✅Every future git push origin main will auto-redeploy on Render.

3 Deploy on Render

1
Sign up at Render
Go to render.com → Sign up with GitHub.
2
New Web Service
Dashboard → New + → Web Service → Connect GitHub → select your dsvv-erp repo.
3
Configure settings
Name: dsvv-erp
Runtime: Python 3
Build Command: pip install -r requirements.txt
Start Command: gunicorn app:app --bind 0.0.0.0:$PORT --workers 2 --timeout 120
Instance Type: Free
4
Add environment variables
Scroll to Environment Variables section → add all variables from the table below → Create Web Service.
5
Wait for first deploy
First build takes 3–5 minutes. Watch the log — you should see Listening on 0.0.0.0:XXXX.
6
Open your live URL
Render gives you a URL like https://dsvv-erp.onrender.com — click it and log in!

4 Environment Variables

In Render Dashboard → your service → Environment tab, add:

VariableValueRequired?
MONGO_URIFull Atlas connection string with passwordRequired
SECRET_KEYLong random string — Render can auto-generateRequired
DB_NAMEdsvv_erpOptional

Generate SECRET_KEY locally

python3 -c "import secrets; print(secrets.token_hex(32))"
# Copy the output → paste into Render's SECRET_KEY field
🔐Never hardcode secrets in app.py. Only set them in Render's environment panel.

5 Verify the Deployment

  1. Open your Render URL → you should see the DSVV ERP login page
  2. Log in with admin / admin123 (auto-created on first boot)
  3. Go to User Management → change the admin password immediately
  4. Add a Course, then a Student, then create a student user account and test login
  5. Add a Teacher, create a teacher user account, test attendance marking
  6. Check Analytics → should show charts after some data is entered
🎉If you see the dashboard — you're live! All data is stored in MongoDB Atlas.

8 Troubleshooting

App crashes on Render

Check Render → your service → Logs tab. Common causes:

  • MONGO_URI not set → add it in Environment tab, redeploy
  • Wrong Atlas password → special chars must be URL-encoded
  • IP not whitelisted → Atlas → Network Access → add 0.0.0.0/0
  • Import error → check requirements.txt has all packages

Free tier sleep (Render)

Render free tier spins down after 15 min of inactivity. First request after sleep takes ~30 seconds. Upgrade to a paid plan or use UptimeRobot to ping every 10 min.

Push changes

git add .
git commit -m "your change"
git push origin main
# Render auto-deploys in ~2 minutes

Reset admin password

In MongoDB Atlas → Data Explorer → dsvv_erp → users collection → delete the admin document. The app recreates it with admin / admin123 on next boot.
